# Pattern: turn CLI gaps into explicit desk tasks

ID: pill-003

## What

Turn each discovered CLI gap into an explicit tracked task. See [[pill-002]].

## Why

A capability audit is only useful if the discovered gaps become executable work instead of floating notes.

## When

Apply this pill whenever a documented SLDB surface fails during desk or docs execution.

## Where

Applies to desk capability audits and to the follow-up modeling of docs.

## How

Capture the gap as a task with direct file references, expected behavior, and validation steps. Reuse the closing discipline from ![[pill-001]].

## How Not

Do not leave capability gaps only in chat notes or memory.
